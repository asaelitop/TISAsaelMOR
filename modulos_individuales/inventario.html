Inventario

Objetivo

Gestionar de manera eficiente los insumos y productos de la cafetería para que permita un control preciso del stock que se logra mediante la automatización del registro de entradas y salidas, asegurando que siempre haya insumos disponibles para preparar los productos evitando pérdidas y optimizando la rentabilidad.

Usuarios que ocupan el Módulo

  • Administrador
  • Empleado

Requerimientos de Administrador y Empleado

El administrador debe ser capaz de:

    Crear un nuevo producto:

  • Tabla proveedores: nombre_proveedor, apellido_paterno, apellido_materno, RFC, contacto_proveedor.
  • Tabla ingredientes: nombre_ingrediente, unidad_medida, cantidad_disponible, stock_minimo, proveedor_id.
  • Tabla productos: nombre_producto, precio_venta, descripcion, categoria, stock_actual, stock_minimo.
  • Tabla recetas: producto_id, ingrediente_id, cantidad_necesaria.

    leer la informacion registrada:

  • Tabla proveedores: id_proveedor, nombre_proveedor, RFC, contacto_proveedor
  • Tabla ingredientes: id_ingrediente, nombre_ingrediente, unidad_medida
  • Tabla productos: id_producto, nombre_producto, precio_venta, descripcion, categoria
  • Tabla recetas: producto_id, ingrediente_id, cantidad_necesaria.

    Actualizar datos registrados:

  • Tabla proveedores: Se usa id_proveedor para identificar. Se pueden actualizar nombre_proveedor, apellido_paterno, apellido_materno, RFC y contacto_proveedor.
  • Tabla ingredientes: Se usa id_ingrediente para identificar. Se pueden actualizar cantidad_disponible, stock_minimo y proveedor_id.
  • Tabla productos: Se usa id_producto para identificar. Se pueden actualizar precio_venta, descripcion, categoria, stock_actual y stock_minimo.
  • Tabla recetas: Se usan producto_id e ingrediente_id para identificar. Se puede actualizar cantidad_necesaria.

    Eliminiar registros directamente:

  • Tabla proveedores: id_proveedor.
  • Tabla ingredientes: id_ingrediente.
  • Tabla productos: id_producto.
  • Tabla recetas: producto_id e ingrediente_id.

El Empleado debe ser capaz de:

    Leer la informacion registrada:

  • Tabla proveedores: id_proveedor, nombre_proveedor, RFC, contacto_proveedor
  • Tabla ingredientes: id_ingrediente, nombre_ingrediente, unidad_medida
  • Tabla productos: id_producto, nombre_producto, precio_venta, descripcion, categoria
  • Tabla recetas: producto_id, ingrediente_id, cantidad_necesaria.

    Actualizar los datos existentes (mediante la venta de un producto):

  • Tabla ingredientes: Se usa id_ingrediente para identificar. Se pueden actualizar cantidad_disponible, stock_minimo y proveedor_id.
  • Tabla productos: Se usa id_producto para identificar. Se pueden actualizar precio_venta, descripcion, categoria, stock_actual y stock_minimo.
  • Tabla recetas: Se usan producto_id e ingrediente_id para identificar. Se puede actualizar cantidad_necesaria.

    Elminiar los datos existentes (mediante una venta)

  • Tabla ingredientes: id_ingrediente.
  • Tabla productos: id_producto.
  • Tabla recetas: producto_id e ingrediente_id.

    Cancelar pedidos (si es que se comete algun error que lleve a una cancelacion):

  • Tabla recetas: producto_id, ingrediente_id, descripcion
  • Tabla ingredientes: id_ingrediente, descripcion
  • Tabla productos: id_producto, descripcion
