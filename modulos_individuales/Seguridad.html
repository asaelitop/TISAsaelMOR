Seguridad

Objetivo

Proteger la información y accesos del sistema a través de la creación, consulta, modificación y eliminación de usuarios con distintos niveles de permisos con el propósito de garantizar la integridad de los datos y evitar accesos no autorizados al sistema.

Usuarios que ocupan el Modulo

  • Administrador

Requerimientos de Administrador

El administrador debe de ser capaz de:

    CREAR: Para crear un nuevo registro, se utilizan los siguientes atributos de cada tabla:

  • Tabla usuarios: nombre, apellido_paterno, apellido_materno, nombre_usuario, contrasena_hash, email, fecha_creacion, activo.
  • Tabla roles: nombre_rol, descripcion.
  • Tabla usuario_rol: id_usuario, id_rol.

    LEER: Solo visualizar los datos registrados si se desea :

  • Tabla usuarios: id_usuario, nombre, fecha_creacion
  • Tabla roles: id_rol, nombre_rol
  • Tabla usuario_rol: id_usuario_rol, id_usuario, id_rol

    ACTUALIZAR: Para modificar un registro existente, se usan los siguientes atributos::

  • Tabla usuarios: Se utiliza id_usuario para identificar el registro. Se pueden actualizar los atributos: nombre, apellido_paterno, apellido_materno, contrasena_hash, email, activo.
  • Tabla roles: Se utiliza id_rol para identificar el registro. Se pueden actualizar los atributos: nombre_rol, descripcion.
  • Tabla usuario_rol: iliza id_usuario_rol para identificar el registro. Se pueden actualizar los atributos: id_usuario, id_rol.

    ELIMINAR: Para eliminar un registro de la base de datos, se usan los siguientes atributos:

  • Tabla usuarios: id_usuario.
  • Tabla roles: id_rol.
  • Tabla usuario_rol: id_usuario_rol, id_usuario, id_rol
